IsochronesRequest

User's request for isochrones

FieldTypeLabelDescription
max_cost double optional

Max cost restrictions for single isochrone. Should be in range [0,+Inf]. Minumim is 0. Example: 2100.0

max_nearest_radius double optional

Max radius of search for nearest vertex (Optional, default is 25.0, should be in range [0,+Inf]) Example: 25.0

lon double

Longitude Example: 37.601249363208915

lat double

Latitude Example: 55.745374309126895

GPSToMapMatch

Representation of GPS data

FieldTypeLabelDescription
tm string

Timestamp. Field would be ignored for request on '/shortest' service. Example: 2020-03-11T00:00:00

lon double

Longitude Example: 37.601249363208915

lat double

Latitude Example: 55.745374309126895

accuracy double optional

GPS measurement accuracy in meters (optional, <=0 or null means use default sigma) Example: 5.0

IntermediateEdge

Edge which is not matched to any observation but helps to form whole travel path

FieldTypeLabelDescription
geom GeoPoint repeated

Edge geometry as line feature

weight double

Travel cost Example: 2.0

id int64

Edge identifier Example: 4278

MapMatchRequest

User's request for map matching

FieldTypeLabelDescription
max_states int32 optional

Max number of states for single GPS point (in range [1, 10], default is 5). Field would be ignored for request on '/shortest' service. Example: 5

state_radius double optional

Max radius of search for potential candidates (in range [7, 50], default is 25.0) Example: 7.0

gps GPSToMapMatch repeated

Set of GPS data

MapMatchResponse

Server's response for map matching request

FieldTypeLabelDescription
sub_matches SubMatch repeated

Array of sub-matches (segments split when route cannot be computed between consecutive points)

warnings string repeated

List of warnings

ObservationEdge

Relation between observation and matched edge

FieldTypeLabelDescription
obs_idx int32

Index of an observation. Index correspondes to index in incoming request. If some indices are not presented then it means that they have been trimmed Example: 0

edge_id int64

Matched edge identifier Example: 3149

vertex_id int64

Matched vertex identifier Example: 44014

matched_edge GeoPoint repeated

Corresponding matched edge as line feature

matched_edge_cut GeoPoint repeated

Cut for excess part of the matched edge. Will be null for every observation except the first and the last. Could be null for first/last edge when projection point corresponds to source/target vertices of the edge

matched_vertex GeoPoint

Corresponding matched vertex as point feature

projected_point GeoPoint

Corresponding projection on the edge as point feature

next_edges IntermediateEdge repeated

Set of leading edges up to next observation (so these edges is not matched to any observation explicitly). Could be an empty array if observations are very close to each other or if it just last observation

SubMatch

A single continuous matched segment

FieldTypeLabelDescription
observations ObservationEdge repeated

Set of matched edges for observations in this segment

probability double

Probability from Viterbi algorithm for this segment Example: -86.578520
